# CMC Markets Bermuda — BTC Price Lock (Event Build V4)

V4 tightens the event-night UX and is ready for a public hosted rollout.

## What changed in V4

- manual-only registration with browser autofill kept for fast entry
- first name + last initial now auto-fills the public display name by default
- premium error handling that scrolls guests to the first missing / invalid field
- cleaner confirmation screen with simpler leverage explanations
- winner icon changed to a gold trophy instead of a dark scooter emoji
- improved mobile dashboard header layout
- cleaner leaderboard fit on venue screens
- built-in QR generation from your hosted public URL
- Hostinger VPS deployment files included:
  - `docker-compose.yml`
  - `Caddyfile`
  - `HOSTINGER_DEPLOYMENT.md`

## The production recommendation

For the live event, host this on a **public HTTPS domain** such as:

- `https://play.yourdomain.com/`

That makes the QR code, registration page, and dashboard accessible from anywhere.

## Environment variables

Copy `.env.example` to `.env` and set at least:

- `PUBLIC_DOMAIN`
- `CMC_ADMIN_PASSWORD`
- `CMC_SESSION_SECRET`
- `CMC_PUBLIC_BASE_URL`
- `CMC_HTTPS_ONLY=true`

Optional:

- `CMC_LEAD_EXPORT_EMAIL`
- `SMTP_HOST`
- `SMTP_PORT`
- `SMTP_USER`
- `SMTP_PASS`
- `SMTP_FROM`

## Local preview (Windows)

1. Extract the ZIP.
2. Open the project folder.
3. Double-click `setup_windows.bat`
4. When setup finishes, double-click `run_windows.bat`

Then open:

- `http://127.0.0.1:8000/`
- `http://127.0.0.1:8000/dashboard`
- `http://127.0.0.1:8000/admin`

## Local preview (Mac / Linux)

```bash
cd cmc_btc_price_lock
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
./run.sh
```

## Production with Docker

```bash
docker compose up -d --build
```

This stack uses:

- the Python app
- Caddy as the HTTPS reverse proxy

## Important notes

- The QR code is generated by the app itself and points to `CMC_PUBLIC_BASE_URL`.
- On a fresh start, the package does **not** include old lead data.
- If you want a completely clean reset on an existing install, delete `data/app.db` before launching.
- Health check endpoint: `/healthz`
- Full Hostinger deployment steps: `HOSTINGER_DEPLOYMENT.md`
